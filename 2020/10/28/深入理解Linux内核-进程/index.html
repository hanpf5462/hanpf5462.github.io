进程

    进程、轻量级进程和线程

  • 进程：程序执行时的一个实例，可以看作充分描述程序已经执行到何种程度的数据结构的汇集。
  • 进程有自己独立的堆栈，所以子进程对内存单元的修改，对父进程是不可见的。
  • Linux使用轻量级进程，对多线程应用进行支持，即共享地址空间、打开的文件等，其中一个修改，另一个可以立即看到。

    进程描述符

  • 包含一个进程相关的所有信息，task_struct类型结构体。
  • 进程描述符

    进程状态

  • 可运行状态（TASK_RUNNING）
    • 正在CPU执行或准备执行
  • 可中断状态（TASK_INTERRUPTIBLE）
    • 进程挂起直到某个条件。中断或信号唤醒都可以。
  • 不可中断状态（TASK_UNINTERRUPTIBLE）
    • 信号不可改变状态，一些特殊情况会用到。如进程打开某个设备文件，驱动探测设备时会用到，完成前都不可被中断，否则可能会处于不可预知状态。
  • 暂停状态（TASK_STOP）
    • 进程的执行被暂停，SIGSTOP、SIGTSTP、SIGTTIN、SIGTTOU信号，进入暂停状态
  • 跟踪状态（TASK_TRACED）
    • 进程被debugger暂停，当一个进程被另一个进程监控时，如ptrace()监控测试程序，任何信号都可以把这个进程置为TASK_TRACED。
  • 僵死状态（EXIT_ZOMBIE）
    • 子进程退出，但父进程未回收资源，处于僵死状态，可能父进程会需要。可以使用wait4()或waitpid()回收资源。
  • 僵死撤销状态（EXIT_DEAD）
    • 防止其他线程在同一进程上执行wait()类似回收，将僵死状态改为僵死撤销状态，防止竞争。
  • state字段赋值： 
    p->state = TASK_RUNNING;

set_task_state()
set_current_state()

标识一个进程

  • 进程和进程描述符一一对应，内核对进程的大部分引用，是通过进程描述符指针进行的。
  • PID，可以修改/proc/sys/kernel/pid_max来限制PID上限。
  • pidmap-array位图，来表示已分配和闲置PID，存放在单独页中。
  • 线程组是相同的PID，存在于进程描述符的tgid字段中。getpid()，返回的是当前进程的tgid，不是PID，想爱你承租的tgid与pid相同。

    进程描述符处理

  • 对每个进程，Linux把两个结构体存放在一个单独位进程分配的存储区内：
    • 内核态的进程堆栈
    • 线程描述符
  • 2page 8k页框存放两种数据结构的方式
    • thread_info占用52字节，所以内核栈能扩展到8140。
      /* 进程的线程描述符和堆栈 */
union thread_union{
    struct thread_info thread_info;
    unsigned long stack[2048];
}
    • 内核使用 alloc_thread_info和free_thread_info宏分配和释放存储thread_info结构和内核栈的内存区。

      标识当前进程

  • 获取当前进程的指针 
    current_thread_info()
/* 执行如下汇编操作 */
movl $0xfffe000,%ecx /* 或者是用于4k堆栈的0xfffff000 */
andl %esp,%ecx
movl (%ecx),p
    运行完成后，p就是当进程的描述符指针，current宏，本质上等同于调用current_thread_info()->task

    TASK_RUNNING状态的进程链表

  • 早期的进程调度，单纯的遍历运行队列；现在通过优先权重和CPU队列拆分，提高调度程序的操作效率。
  • prio_array_t数据结构，存放所有进程列表。

  • 程序创建的进程有父/子关系，多个子进程的话，子进程间有兄弟关系。
  • 一组亲属关系的说明，P0创建P1,P2,P3,P3创建P4。

  • pidhash表及链表
    • 一些特殊情况，内核必须能从进程PID导出对应的进程描述符指针。
      • 如，为kill()系统调用时，P1进程希望向P2发送一个信号，P1调用kill()，参数为P2的PID，内核需要从PID导出其对应的进程描述符。然后从P2的进程描述符中取出记录挂起信号的数据结构的指针。
    • 为加速查找，引入4个散列表。
      Hash表类型 字段名 说明
      PIDTYPE_PID pid 进程的PID
      PIDTYPE_TGID tgid 线程组领头进程的PID
      PIDTYPE_PGID pgrp 进程组领头进程的PID
      PIDTYPE_SID session 会话领头进程的PID
    • pid转换索引
      #define pid_hashfn(x) hash_long((unsigned long) x, pidhash_shift)
/* 很多散列函数都使用 hash_long() */
unsigned long hash_long(unsigned long val, unsigned int bits)
{
    unsigned long hash = val * 0x9e370001UL;
    return hash >> (32 - bit);
}

      • pid散列表

        int nr pid数值
        struct hlist_node pid_chain 链接散列表的下一个和前一个元素
